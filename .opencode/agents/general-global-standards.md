# 全栈通用核心准则

## 1. 架构演进与风险分级
在任何全栈决策中，遵循：**安全 > 质量 > 速度**。
- **架构方向**：客户端已演进为 `frontend-v2/`（Next.js 16+ App Router），采用薄 page + `lib/format` / `lib/normalize` / `lib/queries` 解耦（见 `frontend-v2-structure.md`），完全取代旧版 SPA (Vite)。
- **P0 (阻断级)**: 密钥/PII 泄露、SQL 注入、鉴权绕过、数据不可逆丢失、危险命令。
- **P1 (必验级)**: 登录/鉴权逻辑、核心业务流程、外部 API 集成、DB Migration。
- **P2 (建议级)**: UI/文案微调、内部工具、监控指标。

## 2. 核心红线
- **安全**: 严禁明文存储敏感信息（密钥、Token）。严禁字符串拼接构造 SQL。严禁暴露未鉴权的敏感接口。
- **质量**: 严禁重构与逻辑变更混合提交。严禁硬编码环境变量。核心逻辑必须捕获异常并有追踪点。
- **数据库**: 对于数据库表结构变更，必须使用 Alembic。严禁直接在数据库中执行 DDL 语句进行结构变更。


## 3. 语言规范
- **强制中文回复**: 无论用户用何种语言，Agent 必须始终用中文回复、编写文档、注释及 **Git 提交信息**。
- **禁止中英混合标题**: 所有标题必须是纯中文，不使用英文括号备注与英文缩写标题。
- **术语处理**: 必要英文术语请放入正文句子中说明，不进入标题层级。

## 4. 文件路径与操作规范
- **禁止绝对路径**: 严禁在内部文档或链接中使用绝对路径。必须始终使用相对路径。
- **Git 提交**: 提交信息必须使用语义化前缀（如 `feat:`, `fix:`, `docs:`）且必须为中文。

## 5. 接口与数据规范
- **RESTful**: 优先使用 `/api` 前缀。路径必须遵循资源导向原则，**路径中严禁包含任何动词**（如禁止 `/get_user`, `/stream_refine`），必须使用复数名词表示资源（如 `/users`, `/refinement-streams`）。
- **响应一致性**: 保持 JSON 结构稳定。
- **鉴权隔离**: 写操作强制注入 `current_user`。
- **数据清洗收敛**: 后端数据清洗必须在解析/转换工具函数层统一完成，禁止在各渲染组件中做 ad-hoc 修复。
