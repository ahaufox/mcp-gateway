# 🗺️ MCP Gateway 项目开发路线图 (Roadmap)

本文档规划了 MCP Gateway 项目的开发里程碑与关键演进任务，反映了从核心网关重构到生态子服务建设的整体进程。

---

## 📍 阶段 1: 核心网关架构升级 (已完成 - Completed)
> **目标**: 打造企业级高性能、高可用的 MCP 聚合网关核心。

- [x] **统一协议适配层**: 抽象出传输层接口 (`transport.Transport`)，统一适配 stdio, SSE 和 Streamable HTTP 协议。
- [x] **进程守护与生命周期管理**: 构建 `process.ManagedProcess` 进程管理器，实现下游 stdio 服务的异常退出监控、自动重启和优雅退出。
- [x] **系统稳定性防线**:
  - [x] **错误分类与包装**: 定义了网关专用的错误码体系，统一返回标准格式。
  - [x] **分布式熔断隔离**: 引入 `CircuitBreaker` 熔断机制，防止单点故障引发网关整体级联崩溃。
  - [x] **分级超时与重试**: 支持全局及单服务的精细化超时，设计了带指数退避算法的重试中间件。
- [x] **网关核心功能增强**:
  - [x] **工具重命名与命名空间**: 支持工具别名映射、前缀与命名空间注入，彻底解决多个下游服务工具重名冲突问题。
  - [x] **工具调用缓存 (Cache)**：内置基于 LRU + TTL 的缓存机制，减少对下游计算密集型工具的重复调用开销。
  - [x] **并发/串行批量调用**：提供批量工具调用 API，提升 AI Agent 执行组合任务的吞吐率。
  - [x] **转换钩子链 (Hook System)**：支持自定义前置/后置拦截器，用于请求参数注入和响应结果重写。
- [x] **内置 Web 仪表盘 (Dashboard)**：采用 React 19 + TypeScript + Vite 8 + Tailwind CSS 4 构建，支持服务监控、配置在线可视化转换、登录鉴权及变更日志展示。

---

## 📍 阶段 2: 生态服务集成与容器化交付 (进行中 - In Progress)
> **目标**: 实现服务的标准化打包与高内聚交付，扩展网关支持的生态工具库。

- [x] **网关核心容器化**: 编写多阶段构建的 `Dockerfile`，集成 Go 网关二进制与 Node 运行环境，支持开箱即用。
- [x] **Jules AI 代理服务**: 子模块代码已完成集成，编写了独立 Dockerfile 并完美接入网关 `docker-compose.yaml` 进行一键启动。
- [x] **Douyin 视频解析服务**: 新增 `douyin-mcp` 模块，使用 Python FastAPI 构建，实现视频音频下载与文本提取，已完成容器化并配置健康检查。
- [x] **Chart 图表生成服务**: 集成基于 Node.js/AntV 的图表生成子模块，完成本地构建并在主网关中以 npx 方式完美接入。
- [ ] **PyMCPAutoGUI 图形自动化**:
  - [x] **核心功能编写**: OCR 屏幕识别、键鼠模拟及窗口管理开发完毕。
  - [ ] **容器化环境适配**: 正在克服 Linux 容器下的图形界面依赖（Xvfb 虚拟显示服务器、EasyOCR 权重下载、xdotool 等图形库打包），目前仍处于调试阶段。
- [x] **网络连通与安全性验证**:
  - [x] 验证多容器在 Docker Compose 桥接网络下的互联互通。
  - [x] 引入基于多 Token 的 Bearer 鉴权，支持通过环境变量安全传递凭据。

---

## 📍 阶段 3: 多租户与动态凭据管理 (规划中 - Planned)
> **目标**: 支持多用户空间隔离，实现灵活的安全凭据动态配置与加密持久化。

- [ ] **多租户路由隔离**: 构建基础的多租户架构，实现不同用户的调用空间、配额和会话的彻底隔离。
- [ ] **动态凭据配置管理**:
  - [ ] **服务端凭据加密存储**: 废弃在配置文件中硬编码或通过系统环境变量明文注入第三方密钥（如 GitHub Token、Notion Key 等）的做法，引入加密数据库（如 SQLite + AES-GCM 或外部 Vault 方案）。
  - [ ] **Web 凭据配置面板**: 支持用户在 Web 仪表盘动态配置、更新和删除个人的下游服务 API 令牌。

---

## 📍 阶段 4: 网关高级控制与动态管理 (规划中 - Planned)
> **目标**: 进一步提升网关的易用性、可观测性及传输性能。

- [ ] **服务动态热插拔**: 实现无需重启 Go 网关进程，即可通过 Web 控制面板或 API 动态加载、卸载或修改下游 MCP 服务器配置。
- [ ] **传输性能增强**:
  - [ ] **WebSocket 全面适配**: 进一步打通底层 WebSocket 客户端在核心网关 Client 中的路由与连接池集成，提供高并发双向长连接。
  - [ ] **连接池管理器 (Connection Pool)**: 实现底层的连接池机制，重用 SSE 和 TCP 链路，降低长连接频繁建立的消耗。
  - [ ] **gRPC 协议适配**: 探索引入 gRPC 协议传输层，为超大数据传输和微服务级联提供极速通道。
- [ ] **仪表盘监控可视化升级**: 在 Web 面板展示服务健康状态指标（如 QPS 趋势图、熔断器触发状态、响应延迟等）的实时折线图。
