# FastAPI 高可扩展 API 开发规范 (Scalable API)

## 核心原则
- **简洁技术响应**: 提供准确的 Python 示例，保持代码简洁。
- **函数式编程**: 优先使用纯函数和声明式模式，避免不必要的类。
- **模块化**: 优先考虑代码的可复用和模块化，减少重复。
- **描述性变量**: 使用 `is_active`, `has_permission` 等带谓词的变量名。
- **命名规范**: 目录和文件使用小写下划线 (如 `routers/user_routes.py`)。

## 代码风格
- **异步优先**: 纯函数使用 `def`，涉及 I/O 或异步操作使用 `async def`。
- **类型提示**: 所有函数签名必须包含 Type Hints，优先使用 Pydantic 模型进行验证。
- **RORO 模式**: 采用 "Receive an Object, Return an Object" 模式，增强接口兼容性。

## 异常处理与验证
- **防御性编程**: 在函数开始处处理错误和边缘情况。
- **尽早返回**: 使用 Early Returns 避免深层嵌套。
- **Guard Clauses**: 使用卫语句处理前置条件。
- **自定义异常**: 使用自定义错误类型或工厂保持错误处理的一致性。

## 性能优化
- **非阻塞 I/O**: 对所有数据库调用和外部 API 请求执行异步操作。
- **缓存策略**: 使用 Redis 或内存存储缓存高频访问数据。
- **延迟加载**: 对大数据集使用 Lazy Loading 技术。

## 最佳实践
- **依赖注入**: 充分利用 FastAPI 的 DI 系统管理状态和共享资源。
- **生命周期管理**: 优先使用 `lifespan` 上下文管理器，而非旧版的 `startup`/`shutdown` 事件。
- **中间件**: 合理使用中间件处理日志、错误监控和性能优化。
