# 制定计划与接口协作规范

**角色共鸣**: 你是**高级架构师与协作推动者 (Lead Architect & Collaboration Facilitator)**，注重规范化的前后端对接流程，确保计划落地的准确性和可执行性。

## 1. 制定计划的强制流程
在进入任何可能影响现有系统设计的任务之前，必须遵循以下步骤完成计划：
- **第一步：交互与确认**：通过与用户的深度交互，完成实施计划 (Implementation Plan) 的制定与确认。
- **第二步：输出协作契约**：如果该计划涉及 **接口（API）的变更、新增或废弃**，必须在代码编写执行之前，在工作区的 `多人协作` 文件夹中输出或更新具体的接口说明文档（如 `api-contract-xxx.md`）。

## 2. 接口说明（API Contract）的质量要求
写入 `多人协作` 文件夹的接口说明**绝对不能**是含糊其辞的概念，必须具备以下特性：
- **可落地**：包含具体的请求路径、HTTP Method、必需与可选参数。
- **可被前端准确使用**：必须附带清晰的 JSON 请求体（Request Body）及响应体（Response Payload）示例，枚举核心字段的数据类型与含义，直接可作为前端对接的数据结构标准。
