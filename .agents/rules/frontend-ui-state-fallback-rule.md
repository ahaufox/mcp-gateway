# UI 状态兜底与逆向级联规范

## 1. 列表兜底 (List Fallback)

- **强约束**: 当渲染列表（如记录、卡片、阶段步骤）时，必须针对空数组 `[]` 或 `undefined` 的情况提供明确的兜底 UI（如“暂无数据” 或 “No data”），严禁渲染出空白卡片或引起页面崩溃。
- **示例**:
  ```tsx
  {items && items.length > 0 ? (
    items.map(item => <Card key={item.id} data={item} />)
  ) : (
    <EmptyState message="暂无数据" />
  )}
  ```

## 2. 逆向级联逻辑 (Reverse Cascading Completion)

- **场景定义**: 在逐步进阶的 UI 组件中（如 Workbench 多阶段进度、向导表单等），如果最后一个阶段被标记为完成或包含数据，那么逻辑上必须确保其前面的所有前置阶段也在视觉上呈现“已完成”状态（例如：背景色置灰、勾选等）。
- **强约束**: 必须实现逆向级联的检查与修正，不能仅依赖数组是否有数据，因为即使中间过程数据为空，如果结果已产出，状态树也应当一致闭环。
- **验证**: 对于状态机的任何更新，必须有一个通用的检查机制来校验上游阶段的状态。

## 3. 思维链引导 (CoT)

- **触发条件**: 当开发或修改任何涉及阶段流转、多步引导、空状态展示的组件时。
- **操作**: 必须先使用 `<thinking>分析空状态兜底与级联依赖树...</thinking>` 输出你的逻辑链，随后再进行编码。
