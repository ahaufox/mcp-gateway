# Django 开发最佳实践 (Django Standards)

## 核心原则
- **利用框架特性**: 优先使用 Django 内置功能，避免重新发明轮子。
- **PEP 8 合规**: 严格遵循 PEP 8 风格指南，确保代码可读性。
- **模块化应用**: 使用 Django Apps 结构构建项目，促进关注点分离。

## 开发规范
- **视图选择**: 复杂逻辑使用类视图 (CBVs)，简单逻辑使用函数视图 (FBVs)。
- **ORM 优先**: 优先使用 Django ORM 进行数据库交互，避免原生 SQL。
- **模型验证**: 利用 Django 的 Form 和 ModelForm 类进行数据处理和校验。
- **MVT 模式**: 严格遵守 Model-View-Template 模式。

## 性能优化
- **查询优化**: 使用 `select_related` 和 `prefetch_related` 减少数据库查询次数。
- **缓存框架**: 配置 Redis 或 Memcached 减少数据库负载。
- **异步任务**: 对 I/O 密集型或耗时操作使用 Celery + Redis。

## 安全性
- **内置防护**: 启用并正确配置 CSRF, SQL 注入防护, XSS 预览等安全特性。
- **加密存储**: 遵循 Django 安全建议，确保敏感信息（如密码）正确加密。

## 异常处理
- **视图层处理**: 在视图层捕获并处理异常。
- **自定义错误页**: 定制 404/500 页面以提升用户体验。
- **信号解耦**: 使用 Django Signals 解耦错误处理与核心业务逻辑。
